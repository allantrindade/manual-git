Controle de Versão

O controle de versão é um sistema que registra as mudanças feitas em um arquivo ou um conjunto de arquivos ao longo do tempo, de forma que você possa recuperar versões específicas. Você pode usar um controle de versão para compartilhar arquivos versionados, realizar trabalhos em equipes e controlar as modificações feitas, o sistema de controle de versão está dividido em três partes (Controle de Versão Locais, Controle de Versão Centralizados e Controle de Versão Distribuídos).O sistema de controle local é um mecanismo simples onde você salva suas versões do seu sistema ou programa, na sua própria máquina, esse sistema começou com um processo manual onde você salvava até mesmo em pastas, quando você iria fazer alguma alteração dele, criava uma cópia daqueles arquivos de uma forma manual, a partir disso surgiu o sistema de controle de versão locais que era sistemas simples para você controlar o seu código localmente então esse seria um controle de versão local. O sistema de controle de versão centralizados é um sistema onde é possível trabalhar em conjunto com outros desenvolvedores, através de um servidor central onde está hospedado todas as versões do sistema ou programa e os outros desenvolvedores tem acesso a essas versões através do servidor central. O sistema de controle de versão distribuído, cada desenvolvedor tem a opção de copiar as versões completas do repositório de cada um, com esse sistema você pode colaborar com projetos de outras pessoas, trabalhando com eles de uma forma simples, rápida e segura, o GIT utiliza o sistema de controle de versão distribuído.
